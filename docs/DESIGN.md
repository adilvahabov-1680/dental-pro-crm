## 1. Дизайн-направление

**Premium medical-tech.** Не «админка с таблицами», а современная медицинская платформа: тёмный navy-интерфейс, cyan/teal-акценты, мягкие градиенты, скруглённые карточки, лёгкая футуристичность. Спокойный, чистый, без перегруза — врач работает в нём весь день.

**Анти-цели:** дешёвые тени, кричащие цвета, плотные таблицы без воздуха, хаотичные иконки из разных наборов, переанимированность.

## 2. Цветовые токены

Все цвета — только через CSS-переменные / Tailwind-токены. Никаких hex в компонентах.

| Токен | Значение | Использование |
|-------|----------|---------------|
| `--bg-base` | `#0B1220` (deep navy) | фон приложения |
| `--bg-surface` | `#111B2E` | карточки, панели |
| `--bg-elevated` | `#16233B` | модалки, поповеры, hover-слой |
| `--border-subtle` | `#1E2D47` | границы карточек, разделители |
| `--accent` | `#22D3EE` (cyan) | primary-действия, активные состояния, фокус |
| `--accent-deep` | `#0E7490` (teal) | градиенты, вторичные акценты |
| `--gradient-accent` | `linear-gradient(135deg, #22D3EE → #0E7490)` | primary-кнопки, выделенные карточки, логотип-зона |
| `--text-primary` | `#E6EDF7` | основной текст |
| `--text-secondary` | `#8FA3BF` | подписи, метаданные |

Светлая тема — **не в MVP**, но токены делают её добавление тривиальным.

## 3. Типографика и сетка

- Шрифт: **Inter** (latin-ext — обязателен для азербайджанских ə, ğ, ş, ç, ö, ü, ı).
- Шкала: 12 / 13 / 14 (базовый) / 16 / 20 / 24 / 30. Заголовки — semibold, не bold.
- Цифры в финансах и номерах зубов — `tabular-nums`.
- Сетка: spacing по 4px-шкале; контент-паддинг страниц 24px; gap между карточками 16–24px.
- Радиусы: карточки `16px`, кнопки/инпуты `10px`, бейджи `9999px` (pill).
- Тени: одна мягкая (`0 8px 24px rgb(0 0 0 / 0.25)`) только для elevated-слоя; карточки разделяются цветом фона и border, не тенями.

## 4. Компонентные правила

- **Иконки — только Lucide**, один набор, stroke 1.5–2, размер 16/20.
- **Карточки (`Card`)** — базовый строительный блок: `--bg-surface` + `--border-subtle` + radius 16.
- **Кнопки:** primary — градиент accent; secondary — surface + border; ghost — текст + hover-слой. Высота 36/40.
- **Статусы — всегда pill-бейджи** с точкой-индикатором цвета (см. карту статусов в DENTAL_CHART.md). Один и тот же статус везде выглядит одинаково.
- **Таблицы** — облегчённые: без вертикальных линий, hover-подсветка строки, зебра не используется; на мобильном превращаются в карточки.
- **Формы:** label сверху, ошибка снизу красным 12px, focus-ring цвета accent.
- **Пустые состояния обязательны**: иконка + 1 фраза + CTA-кнопка. Никогда не показывать голую пустую таблицу.
- **Загрузка** — skeleton-блоки (не спиннеры на весь экран).
- **Анимации:** 150–200ms ease-out на hover/открытие; без parallax, без bounce.

## 5. Layout приложения

```
┌──────────┬─────────────────────────────────────┐
│          │  Topbar: поиск пациента · дата ·    │
│ Sidebar  │  уведомления · профиль врача        │
│ (icons + ├─────────────────────────────────────┤
│  labels, │                                     │
│ collapse)│  Page content (cards on navy)       │
│          │                                     │
└──────────┴─────────────────────────────────────┘
```

- **Sidebar** — тёмный, фиксированный, сворачиваемый до иконок; активный пункт — cyan-индикатор + подсветка.
- **Topbar** — глобальный поиск пациентов (Cmd/Ctrl+K), быстрые действия («Yeni pasiyent», «Yeni qəbul»).
- **Dashboard** — сетка stat-карточек (приёмы сегодня, выручка, долги, новые пациенты) с мягким градиент-акцентом, под ними лента приёмов дня.
- Breakpoints: ≥1280 полный layout, 768–1280 свёрнутый sidebar, <768 — нижняя навигация (отложено после MVP, но layout не должен этому мешать).

## 6. UX-принципы по ролям

- **Врач:** из календаря в карточку пациента — 1 клик; зубная карта и план лечения — первый экран профиля пациента; добавление процедуры ≤ 3 клика.
- **Ресепшн:** календарь — главный экран; создание записи из пустого слота; статус приёма меняется одним кликом.
- **Админ/владелец:** dashboard с финансовой сводкой; всё кликабельно вглубь (drill-down).
- Любая запись (пациент, приём, счёт) открывается в **slide-over панели** справа, где это уместно — без потери контекста страницы.

## 7. Процесс разработки UI (обязательный порядок)

1. Сначала **дизайн-токены + базовые компоненты** (Card, Button, Badge, Input, Table, Modal, SlideOver, EmptyState, Skeleton) — один раз, в `components/ui`.
2. Потом layout (Sidebar, Topbar).
3. Потом страницы — **только из готовых компонентов**. Новый визуальный паттерн = сначала компонент, потом использование.
4. Маленькие шаги: один модуль за итерацию, не трогая чужие файлы без необходимости. Не переписывать работающее.
5. Каждая страница проверяется на: читаемость на navy-фоне, пустое состояние, skeleton, переводы AZ.
